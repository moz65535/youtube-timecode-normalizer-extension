リンク単体テスト

各リンクを右クリックし、「リンクをタイムコード正規化URLにしてコピー」と 「正規化したリンクを開く」を確認します。

下表の期待値は初期設定（変換前形式を維持、si保持、feature除去、list非保持、 崩れた時刻を救済）を前提とします。

通常ページ選択範囲のクリップボード確認

  1. 下の試験用テキスト全体を選択し、右クリックメニューから正規化します。
  2. 画面上の試験用テキストが書き換わっていないことを確認します。
  3. 「正規化後の貼り付け先」へ貼り付け、「一致」と表示されることを確認します。
  4. ページ上で右クリックし、「直前の変更を元に戻す」を実行します。
  5. 「Undo後の貼り付け先」へ貼り付け、「一致」と表示されることを確認します。
通常ページ https://www.youtube.com/watch?v=qhH-azW3LJw&t=17m12s

クリップボード書き込み失敗は、権限が付与された通常環境ではこのページから安定して再現できません。 失敗時に成功扱いしないことは、障害注入を使う自動テストの対象として別途扱います。

textarea直接書き換えの確認

  1. 下のtextarea内で「ここから選択」行の次から「ここまで選択」行の直前までを選択します。
  2. 選択範囲を右クリックし、「選択範囲のタイムコード付きURLを正規化」を実行します。
  3. 3件変更の通知が出て、URLがそれぞれ 1032、1653、2765 になることを確認します。
  4. 先頭、選択境界、末尾の各行が変化していないことを確認します。
  5. 右クリックメニューの「直前の変更を元に戻す」で初期内容へ戻ることを確認します。
Undo確認後や、試験をやり直すときに使用します。

contenteditableの保存対象確認

  1. 下の編集欄にあるURL部分を選択し、右クリックメニューから正規化します。
  2. 画面上のURLが t=1032 へ変わり、inputイベント回数が1以上になることを確認します。
  3. 「内部状態を保存結果へ反映」を押し、保存結果にも t=1032 が表示されることを確認します。
  4. 右クリックメニューの「直前の変更を元に戻す」で、画面と内部状態が変更前へ戻ることを確認します。
